Getting Started 🏁

Prerequisites

Before you begin, ensure you have:

  • ✅ Azure DevOps account with project access
  • ✅ Git installed and configured
  • ✅ Azure CLI installed with DevOps extension
  • ✅ Claude Code CLI installed (optional but recommended)
  • ✅ Your preferred code editor

Step 1: Install Azure CLI and DevOps Extension

$ brew install azure-cli
$ az extension add --name azure-devops

Step 2: Configure Azure DevOps

$ az login
$ az devops configure --defaults organization=https://dev.azure.com/NorthHighland project=agentic-sdlc-platform

Step 3: Clone the Repository

$ git clone https://dev.azure.com/NorthHighland/agentic-sdlc-platform/_git/agentic-sdlc-platform
$ cd agentic-sdlc-platform
💡 Pro Tip: Run cat CLAUDE.md to see the platform's AI instructions and understand how Claude Code works with the codebase.

Step 4: Install Claude Code (Recommended)

🤖 Claude Code Installation

Claude Code is the AI-powered CLI that helps you develop faster:

$ npm install -g @anthropic/claude-code

After installation, configure with your API key:

$ claude code configure

Core Development Workflow 🔄

⚠️ CRITICAL: Always follow the "Work Item First" policy. Never create code changes without first creating a work item in Azure DevOps.

The Complete Workflow

1
Create a Work Item

Start by creating a work item in Azure DevOps for your task:

az boards work-item create \ --title "Add user authentication feature" \ --type "User Story" \ --fields \ "System.State=Active" \ "System.Description=<strong>User Story:</strong><br/>As a user, I want to log in securely...<br/><br/><strong>Description:</strong><br/>Implement JWT-based authentication..." \ "Microsoft.VSTS.Common.AcceptanceCriteria=<strong>Acceptance Criteria:</strong><br/>✅ User can register<br/>✅ User can log in<br/>✅ JWT tokens are generated<br/><br/><strong>Definition of Done:</strong><br/>✅ Code reviewed<br/>✅ Tests passing<br/>❌ Human approval obtained"
2
Create a Feature Branch

Create a branch using the work item ID:

# Assuming work item #1234 was created git checkout -b feature/1234-user-authentication
3
Implement Your Changes

Write your code, following the platform standards. Use Claude Code for assistance:

# Use Claude Code to help implement claude code "implement JWT authentication with login and register endpoints" # Or work directly with your editor code src/auth/
4
Commit Your Changes

Stage and commit your changes with a clear message:

git add . git commit -m "feat: implement JWT authentication with login and register endpoints - Add user registration endpoint with validation - Implement secure login with JWT token generation - Add authentication middleware for protected routes - Include comprehensive test coverage Resolves: AB#1234"
5
Create a Pull Request

Push your branch and create a PR linked to the work item:

# Push the branch git push -u origin feature/1234-user-authentication # Create PR with proper linking az repos pr create \ --title "Feature: Add JWT authentication with login and register" \ --description "$(cat <<'EOF' <strong>🚀 Feature Implementation</strong><br/><br/> <strong>Changes:</strong><br/> • Implemented JWT authentication system<br/> • Added login and register endpoints<br/> • Created authentication middleware<br/> • Added comprehensive tests<br/><br/> <strong>Testing:</strong><br/> • ✅ Unit tests passing (100% coverage)<br/> • ✅ Integration tests completed<br/> • ✅ Manual testing performed<br/><br/> <strong>Resolves:</strong> AB#1234<br/><br/> 🤖 Generated with [Claude Code](https://claude.ai/code) EOF )" # Link work item to PR (MANDATORY) PR_ID=$(az repos pr list --query "[?sourceRefName=='refs/heads/feature/1234-user-authentication'].pullRequestId | [0]" -o tsv) az repos pr work-item add --id $PR_ID --work-items 1234
6
Wait for Review and Approval

STOP HERE! Wait for human review and approval before merging. The PR will be reviewed by:

  • 🤖 Automated PR agents for code quality and compliance
  • 👥 Human reviewers for business logic and architecture
  • ✅ Final approval from project maintainers
🛑 NEVER:
  • Use auto-merge or auto-complete flags
  • Merge without human approval
  • Skip the work item creation step
  • Mix unrelated changes in one PR

Best Practices 🌟

📋
Always Start with a Work Item

Never write code without a work item. This ensures traceability and proper planning.

Remember: Work Item → Branch → Code → PR → Review → Merge
🎯
One PR, One Purpose

Keep PRs focused on a single logical change. Maximum 5 files unless necessary.

  • ✅ Fix authentication bug
  • ❌ Fix auth bug + update docs + refactor utils
🔍
Use Semantic Commit Messages

Follow conventional commits for clear history:

  • feat: New feature
  • fix: Bug fix
  • docs: Documentation
  • refactor: Code refactoring
✅
Test Everything

Aim for 100% test coverage on new code. Use Claude Code to help write comprehensive tests.

claude code "write tests for [your-feature]"
📚
Document as You Go

Update documentation with your changes. Future you (and your team) will thank you!

  • API documentation
  • Code comments for complex logic
  • Update README if needed
🛡️
Security First

Never commit secrets or credentials. Use environment variables and secure storage.

Never commit: API keys, passwords, tokens, connection strings

Branch Naming Conventions

Always include the work item ID in your branch name:

  • ✅ feature/1234-add-user-auth
  • ✅ bugfix/5678-fix-login-error
  • ✅ hotfix/9012-security-patch
  • ❌ feature/add-user-auth (missing work item)
  • ❌ my-feature-branch (no context)

Compliance & Security 🛡️

Compliance Driven Development (CDD)

The platform includes comprehensive compliance automation that ensures your code meets regulatory requirements throughout the development lifecycle.

Supported Compliance Frameworks

🏛️
SOC2 Type II

Automated enforcement of 118 controls across 5 trust service criteria. Real-time validation during development.

🔐
ISO 27001:2022

All 93 Annex A controls implemented with automatic evidence collection and control mapping.

🇪🇺
GDPR

Privacy by Design with data minimization, consent management, and automated data subject rights.

🏥
HIPAA

Healthcare data protection with audit controls, encryption requirements, and access management.

💳
PCI-DSS

Payment card security standards with automated vulnerability scanning and secure coding practices.

🏛️
Additional Frameworks

NIST, CCPA, SOX, FedRAMP, and custom frameworks supported through flexible configuration.

How CDD Works

1
Automatic Detection

The platform automatically detects when you're working with:

  • Personal data (triggers GDPR requirements)
  • Healthcare information (triggers HIPAA controls)
  • Payment data (triggers PCI-DSS validation)
  • Security features (triggers SOC2/ISO controls)
2
Real-time Validation

As you code, the platform validates compliance in real-time:

claude code "check compliance for user data handling"
3
Automated Evidence Collection

Evidence is automatically collected for audits:

  • Configuration snapshots
  • Access logs and audit trails
  • Test results and validations
  • Security scan reports
4
Compliance Reporting

Generate compliance reports for auditors:

claude code "generate SOC2 compliance report for Q4"

Security Best Practices

🔒 Secure Coding Guidelines

  • Never hardcode secrets: Use environment variables or secure vaults
  • Encrypt sensitive data: Platform enforces encryption at rest and in transit
  • Access control: Implement least privilege and role-based access
  • Input validation: Sanitize all user inputs to prevent injection attacks
  • Audit logging: All data access is automatically logged

Privacy Protection

🔐 Data Privacy Examples

Common Scenarios 🎭

Scenario 1: Fixing a Bug

🐛 Bug Fix Workflow
  1. Create Bug Work Item:
    az boards work-item create \ --title "Fix: Login fails with special characters in password" \ --type "Bug" \ --fields \ "System.State=Active" \ "Microsoft.VSTS.Common.Severity=2 - High"
  2. Create branch: git checkout -b bugfix/2345-login-special-chars
  3. Use Claude to investigate: claude code "find the login validation logic and check special character handling"
  4. Fix the issue and test
  5. Create PR with bug fix details

Scenario 2: Adding a New Feature

✨ Feature Development
  1. Discuss requirements with team
  2. Create User Story with detailed acceptance criteria
  3. Use Claude Code for implementation plan:
    claude code "create an implementation plan for adding real-time notifications using WebSockets"
  4. Implement incrementally with tests
  5. Get early feedback through draft PR

Scenario 3: Code Review Feedback

📝 Handling Review Comments

When you receive feedback on your PR:

  1. Address each comment thoughtfully
  2. Use Claude Code for suggestions:
    claude code "the reviewer suggests using dependency injection here. Show me how to refactor this"
  3. Update your code and push changes
  4. Respond to each comment in the PR
  5. Request re-review when ready

Scenario 4: Emergency Hotfix

⚠️ For Production Issues:
  1. Create high-priority bug work item
  2. Branch from main: hotfix/3456-critical-payment-fix
  3. Minimal change to fix the issue
  4. Expedited review process
  5. Deploy immediately after approval
